# 开发现状总览更新模板

以下模板用于后续在用户说“请总结，谢谢你”时，生成下一版开发现状总览：

---

请基于：

1. 当前项目代码现状
2. 当前会话上下文
3. 现役主文件实现情况

更新“开发现状总览文档”。

更新时必须覆盖以下内容：

## 1. 当前项目阶段

- 当前整体处于什么开发阶段
- 主流程是否已经打通
- 目前是骨架阶段、第一版可玩阶段，还是功能扩展阶段

## 2. 已完成的主系统

按系统归纳当前已落地内容，例如：

- 主菜单 / 设置
- 角色创建
- 属性系统
- 玩家主体
- 地图与房间
- 战斗与敌人
- 背包 / 装备 / 宝箱
- 结算系统
- Tooltip / 悬浮说明系统

## 3. 可复用基础设施

必须额外指出哪些模块以后能频繁复用，例如：

- 属性重算系统
- 容器 / 物品系统
- 敌人统一 AI 框架
- 结算互斥系统
- 通用悬浮详情窗系统

## 4. 当前默认现役主文件

明确写出当前应该继续开发的主文件，不要把旧版本文件混进来。

## 5. 仍是占位或第一版的部分

明确哪些内容还未真正完整实装，防止新会话误判。

## 6. 当前开发原则

至少写清：

- 只在现役系统上增量开发
- 不新开平行系统
- 不在功能开发时顺手清旧文件

## 7. 建议下一步方向

根据当前现状给出下一批自然延伸的开发方向。

## 8. 风格要求

- 用中文
- 结构清晰
- 新会话读完后就能继续接手开发
- 不要只写抽象概括，要能反映当前项目真实落地情况
