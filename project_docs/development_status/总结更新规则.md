# 总结更新规则

## 固定触发指令
当用户说：

`请总结，谢谢你`

默认执行以下动作：

1. 读取当前项目现役主文件与最近新增模块
2. 结合当前会话上下文，整理本阶段已完成内容
3. 更新或重写 [开发现状总览.md](/e:/working_VSCODE/2026_4_8_game/project_docs/development_status/开发现状总览.md)
4. 明确写出：
   - 目前已经完成了哪些系统
   - 开发到哪一步
   - 哪些系统以后可以复用
   - 当前默认现役主文件
   - 仍是占位或第一版的部分
   - 建议下一步方向

## 更新原则

- 不只总结聊天内容，要结合当前代码现状
- 优先以“现役系统”为准，不把旧脚本误写成现役入口
- 需要说明哪些模块已做成复用基础设施
- 需要说明哪些部分仍然只是占位或未实装完全
- 如果本轮做了新基础设施，必须额外写入“可复用系统”章节
