默认规则：
- 后续开发优先接入现役系统，不新开平行版本
- 旧文件先保留，不在功能开发时顺手清理
- 需要阶段总结时，更新本文档，而不是把进度散落在聊天记录里

### 4. 统一甲 / 乙 / 丙加成系统

## 当前仍属占位或第一版的部分
- 角色创建虽然已经是正式 7 步，但前世职业内容、职业特性和临时属性倾向仍属第一版平衡方案
- 任务系统仍是占位页
- 法术技能尚未真正接入魔力强度主链路
- 宝物系统的真实内容仍未批量接入统一加成系统
- Buff / Debuff 还没有作为标准加成源批量落地
- 商店、仓库、掉落列表等容器尚未接入统一容器交互体系
- 装备特殊效果字段已预留，但大部分还未实现内容
- 对话系统目前仍是第一版正式实现，商店内容、训练内容、敌对剧情动作还只到统一接口层
- 对话数据目前仍以脚本注册表为主，还没有编辑器或资源化工作流
- 新手开场虽然已经正式收束为女神入口，但后续更完整的世界观分支、第二批正式 NPC、更多开场后的差异化内容还没展开
- 实机交互回归还不足，最近完成的 AI 架构、对话系统、trigger 条件语言，以及装备 / 斩击 / 滚轮回归修复，主要通过 headless smoke test 验证

## 当前开发原则
1. 只往现役系统加，不在旧脚本上修补。
2. 新功能优先复用现有模块，不重复造轮子。
3. 不把属性公式、Tooltip 文本、装备说明分散写在多个页面里。
4. 物品、装备、宝箱、背包优先继续收束到统一模板与容器规则。
5. 功能开发阶段不顺手清旧文件；如果要清理，单独作为清理任务处理。
6. 任何“完全删除文件”的操作，都必须先备份到 `litter_box`。
7. 需要阶段总结时，优先更新本文档。
