# 删除前备份规则

## 目的

为了避免以后再次出现“误删文件后无法精确恢复”的情况，本项目从现在开始执行统一规则：

**任何文件在被完全删除之前，必须先复制一份到项目根目录的 `litter_box/`，然后才能删除原文件。**

---

## 适用范围

本规则适用于当前项目目录 `E:\\working_VSCODE\\2026_4_8_game` 下的所有文件，包括但不限于：

- `game/`
- `character_creation/`
- `main_menu/`
- `tools/`
- `project_docs/`
- 其它项目内脚本、场景、文档、配置文件

---

## 强制执行规则

1. 只要是“完全删除文件”，必须先做备份。
2. 备份目标目录固定为项目根目录下的 [litter_box](/e:/working_VSCODE/2026_4_8_game/litter_box)。
3. 不能先删后想起来补备份，必须先复制，再删除。
4. 如果一次删除多个文件，必须把这一批删除文件都先放进 `litter_box`。
5. 如果不确定文件是否还会被引用，优先不要删；至少先备份。

---

## 推荐操作方式

推荐做法：

1. 先在 `litter_box` 下按日期、任务名或原路径建立备份目录。
2. 把准备删除的文件复制进去。
3. 检查备份文件存在且可读。
4. 再删除原文件。

推荐命名示例：

- `litter_box/2026-04-10_player_menu_ui_v2.gd`
- `litter_box/2026-04-10_cleanup_round_1/game/scripts/example.gd`

如果方便，最好保留原始相对路径，便于以后精确回退。

---

## 说明

- `litter_box` 是“删除前备份箱”，不是正式开发目录。
- 放进 `litter_box` 的文件默认只用于回退、核对、追溯，不代表仍在现役主链路中使用。
- 本规则是项目级约定，后续任何人处理删除操作时都应遵守。
